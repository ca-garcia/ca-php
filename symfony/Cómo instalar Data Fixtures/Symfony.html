Como se puede instalar DoctrineFixturesBundle en Symfony 2.3.0 sin utilizar git o composer?

Esta pregunta hace ya varios días que la estaba buscando en Internet sin encontrar respuesta, y hoy probando encontré la solución. Soy nuevo en esto de Symfony, así que pido disculpas de antemano si me equivoco en algo al explicar la solución que encontré.

Estoy trabajando en la documentación, nada mas que termine la publico para que puedan verla.
Saludos

Como lo prometido es deuda, aquí les dejo la documentación sobre como instalar el Bundle DoctrineFixturesBundle en Symfony 2.3.0 sin utilizar git o composer.

1. Descargar el bundle DoctrineFixturesBundle desde git.


2. El bundle DoctrineFixturesBundle utiliza el bundle DoctrineDataFixtures, descargar el bundle DoctrineDataFixtures en git.

3. Se descomprimen los dos bundles.

4.  Los bundles deben quedar con los siguientes nombres:
    Nombre del Bundle           Nombre al descomprimir               Cambiar nombre de la carpeta a
    DoctrineFixturesBundle      DoctrineFixturesBundle-master        doctrine-fixtures-bundle
    Doctrine Data Fixtures       data-fixtures-master                      data-fixtures

5. Importante.
    Las carpetas donde están los bundles deben quedar con el nombre que esta en color rojo.

6. La carpeta que contiene el bundle DoctrineFixturesBundle, que después de cambiado el nombre se llama doctrine-fixtures-bundle, se debe configurar de la siguiente manera.

Crear un directorio Doctrine/Bundle/FixturesBundle dentro de la carpeta doctrine-fixtures-bundle.

 
Fig 1. Directorio  Doctrine/Bundle/FixturesBundle dentro doctrine-fixtures-bundle.

Cortar lo que hay en doctrine-fixtures-bundle y pegarlo en el directorio creado anteriormente (Se corta todo lo que esta señalado en la figura 2 y se pega en el directorio de la figura 3).

 
Fig 2. Carpeta doctrine-fixtures-bundle.
 
 
Fig 3. Directorio Doctrine/Bundle/FixturesBundle.

7. Copiar los dos bundles para el directorio vendor/doctrine/ de su proyecto Symfony 2.3.0

 
Fig 3. Directorio vendor/doctrine/. 

8. Registra el bundle DoctrineFixturesBundle en el archivo AppKernel.php

        // ...
        public function registerBundles()
        {
            $bundles = array(
                // ...
                new Doctrine\Bundle\FixturesBundle\DoctrineFixturesBundle(),
                // ...            );
            // ...
        }

9. Registra los dos bundles en el fichero autoload_namespaces.php que se encuentra en el directorio vendor/composer

    // ...
    return array(
        // ...
       
        'Doctrine\\Common\\DataFixtures\\' => array($vendorDir . '/doctrine/data-fixtures/lib'),
        'Doctrine\\Bundle\\FixturesBundle\\'=>array($vendorDir . '/doctrine/doctrine-fixtures-bundle'),
       
        '' => $baseDir . '/src',
    );

10. Para asegurarse de que la instalación ha sido correcta, ejecuta el siguiente comando para listar todos los comandos de la aplicación y comprueba que se muestra el nuevo comando doctrine:fixtures:load.

11. Si se muestra el comando doctrine:fixtures:load quiere decir que el bundle esta correctamente instalado.


Otro problema que encontré, fue cuando trataba de acceder desde una plantilla twig a un controlador en Symfony 2.3.0

1. Antes se hacía así:

{% render "CiudadBundle:Default:listaCiudades" with {'ciudad': ciudadSeleccionada } %}

2. Pero cuando refrescaba la página me daba el siguiente error:

An exception has been thrown during the rendering of a template ("No route found for "GET Default:listaCiudades"") in "OfertaBundle:Default:portada.html.twig"

3. Si se fijan bien el error lo que quiere decir es que no encuentra el metodo listaCiudadesAction del controlador Default en el Bundle OfertaBundle. En realidad el problema está en que lo busca donde no debe, está buscándolo dentro del Bundle de Oferta, y claro, en realidad está dentro del bundle CiudadBundle.

4. Cambiar esta línea.

Antes:
{% render "CiudadBundle:Default:listaCiudades" with {'ciudad': ciudadSeleccionada } %}
Cambiar a:
{{ render (controller("CiudadBundle:Default:listaCiudades" , { 'ciudad': ciudadSeleccionada })) }}
